# Radar-format messages — automatic send on VM

The **radar format** (ACTIVE RADAR + GNI STRATEGIC READ + Monitoring Status) is generated by `gni/analysis/llm_formatter.py` and sent via `scripts/send_radar_messages.py`. To run it **on the VM** and have it **run automatically**, use the worker container and cron.

## 1. Run on the VM (manual)

From the VM, in the repo directory (e.g. `/opt/gni-bot-creator`):

```bash
# One message, actually send to Telegram
docker compose exec -T worker python scripts/send_radar_messages.py --count 1 --send

# Dry-run (no send)
docker compose exec -T worker python scripts/send_radar_messages.py --count 1
```

Requires in `.env`:

- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_TARGET_CHAT_ID` (or `TELEGRAM_CHAT_ID`)

The **worker** service has the full repo (including `gni/`) and these env vars, so the script runs there.

## 2. Run automatically (cron)

On the VM:

1. Make the runner script executable (once):

   ```bash
   chmod +x /opt/gni-bot-creator/scripts/run_radar_send.sh
   ```

2. Open crontab:

   ```bash
   crontab -e
   ```

3. Add a line (adjust path if your repo is elsewhere):

   ```cron
   # Every 6 hours: send 1 radar-format message to Telegram
   0 */6 * * * /opt/gni-bot-creator/scripts/run_radar_send.sh >> /var/log/gni-radar.log 2>&1
   ```

   Or every day at 09:00:

   ```cron
   0 9 * * * /opt/gni-bot-creator/scripts/run_radar_send.sh >> /var/log/gni-radar.log 2>&1
   ```

   To send 2 messages per run:

   ```cron
   0 */6 * * * /opt/gni-bot-creator/scripts/run_radar_send.sh 2 >> /var/log/gni-radar.log 2>&1
   ```

4. Ensure the log file is writable (e.g. create it and chmod, or use a path in the repo):

   ```bash
   sudo touch /var/log/gni-radar.log
   sudo chown "$(whoami)" /var/log/gni-radar.log
   ```

After that, the system will send radar-format messages on the schedule you chose.
